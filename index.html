OKX 链 AI 助手 · 白皮书

本项目是面向 OKX Chain 的智能链上助手，聚合链上数据、AI 推理与消息机器人能力，帮助用户完成代币分析、合约生成、钱包监控与多语言交互。

愿景与目标

  • 用 AI 降低链上分析门槛，让普通用户也能读懂复杂数据。
  • 在合规安全的前提下，提供一站式工具链与自动化工作流。
  • 通过机器人与 API，连接生态伙伴与开发者。

定价与套餐

  • 免费层：基础查询与有限次数的 AI 对话，适合个人体验。
  • 专业层：更高配额、队列优先与高级端点访问，适合重度用户与交易者。
  • 企业层：专属限流池、SLA、私有部署选项与定制集成。

数据与隐私

  • 仅在必要时存储最小化数据（如请求计数、错误日志），不保存敏感私钥。
  • 遵循“可删除”原则：用户可按需删除其会话与监控任务。
  • 对外调用第三方服务（如 LLM、RPC）时，会进行最小字段透传与脱敏。

安全设计

  • 多层鉴权：API Key、可选 OAuth、Telegram 用户绑定。
  • 细粒度限流与隔离：按 Key、IP 与端点做配额与桶隔离。
  • 提示词与输出审计：高风险操作需显式确认与模板化执行。
  • 可观测性：结构化日志、请求追踪与异常告警。

社区与贡献

欢迎通过 Issue/PR 贡献能力插件、数据适配器与语言包；也可参与提示词与样例库建设。

常见问题（FAQ）

如何集成到我自己的机器人？

参考 API 文档与示例，使用 `/api/*` 端点对接；生产建议配置 Webhook 并启用鉴权与限流。

是否支持多链？

当前以 OKC 为主，计划逐步接入更多 EVM 兼容链，并通过适配层扩展数据源。

我的数据会用于模型训练吗？

默认不会。仅在用户授权的前提下，匿名化聚合数据可用于质量改进与指标统计。

核心能力

  • 代币分析：输入合约地址，自动抓取链上信息并生成 AI 叙事。
  • 智能合约与 NFT：根据自然语言生成代码/元数据。
  • 钱包监控：买入事件识别与 Telegram 推送。
  • 多语言：中英双语，接口层可扩展更多语言。

系统架构

系统采用前后端分离 + Firebase Functions 的无服务器架构，集成 OKX RPC、OpenAI、Telegram Bot 与 Realtime Database。

Clients (Web / Telegram)
   │
   ├── HTTPS → Firebase Functions (Express)
   │     ├── Auth & Rate Limit
   │     ├── OpenAI Service (chat, generate)
   │     ├── OKX RPC Service (chain stats, token info)
   │     ├── Wallet/Tweet Monitor Services
   │     └── Telegram Integration (Polling/Webhook)
   │
   └── Firebase (Firestore/RTDB) for storage & realtime

API 与安全

  • 鉴权：自定义 API Key（`x-api-key` 或 `Authorization: Bearer`）。
  • 限流：按套餐（月配额）统计与限制。
  • 端点：`/api/chat`、`/api/token/analyze`、`/api/chain/analyze`、`/api/wallet/:addr` 等。
POST /api/token/analyze
Headers: { "x-api-key": "" }
Body: { "contractAddress": "0x..." }
→ 200 { success: true, data: { tokenData, chainData, narrative } }

Telegram 集成

机器人同时支持轮询与 Webhook 两种形态。生产推荐使用 Webhook，降低网络不确定性。
